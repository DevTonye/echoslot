Authentication

Echoslot API uses token-based authentication. Include the token in the Authorization header of your requests.

Getting Your API Token

To obtain an API token:

  1. Log into your Echoslot account
  2. Navigate to Settings â†’ API Keys
  3. Generate a new API token
  4. Include it in your requests as shown below
# Include token in request headers
curl -H "Authorization: Token your-api-token-here" \
    -H "Content-Type: application/json" \
    https://api.echoslot.com/api/providers/

Service Providers

Manage service provider profiles, search for providers, and retrieve availability information.

GET /api/providers/

Retrieve a list of service providers. Supports search and filtering.

Query Parameters

Parameter Type Description
search string Search providers by name, bio, or address
ordering string Order results by fields (first_name, last_name)
name string Filter by service name
location string Filter by location/address

Example Response

{
  "count": 42,
  "next": "https://api.echoslot.com/api/providers/?page=2",
  "previous": null,
  "results": [
    {
      "id": 1,
      "first_name": "John",
      "last_name": "Doe",
      "bio": "Experienced service provider...",
      "services": [...]
    }
  ]
}
GET /api/providers/{id}/availability/

Get available time slots for a specific provider and service.

Query Parameters

Parameter Type Required Description
service integer Required Service ID to check availability for

Example Response

{
  "provider": 1,
  "service": 5,
  "available_slots": {
    "2024-01-15": [
      {
        "time": "09:00:00",
        "available": true
      }
    ]
  }
}

Services

Manage services offered by providers, including pricing, duration, and descriptions.

GET /api/services/

Retrieve a list of available services with search and filtering capabilities.

Query Parameters

Parameter Type Description
search string Search services by name or description
provider integer Filter by provider ID
price number Filter by maximum price
duration number Filter by maximum duration

Appointments

Create, manage, and track appointments with comprehensive status handling.

POST /api/appointments/

Create a new appointment. Requires service ID, date, and time.

Request Body

Parameter Type Required Description
service integer/string Required Service ID or URL
appointment_date string Required Date in YYYY-MM-DD format
start_time string Required Time in HH:MM:SS format
first_name string Required Client's first name
last_name string Required Client's last name
email string Required Client's email
phone string Required Client's phone number
notes string Optional Additional notes

Custom Endpoints

The appointments API also includes these useful endpoints:

  • GET /api/appointments/today/ - Today's appointments
  • GET /api/appointments/upcoming/ - Upcoming appointments
  • GET /api/appointments/past/ - Past appointments

Error Codes

Common HTTP status codes and error responses you might encounter.

Status Code Description Possible Causes
400 Bad Request Invalid input data, missing required fields
401 Unauthorized Missing or invalid authentication token
403 Forbidden Insufficient permissions for the action
404 Not Found Resource doesn't exist
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Server-side issue
